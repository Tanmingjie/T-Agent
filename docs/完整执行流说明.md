# 方案设计:用例自动执行的完整链路(16 步)

> 本方案把"一条业务测试用例"从 Excel 一路自动化到产出可维护测试代码,
> 核心思路是 **AI 负责"怎么操作",规则引擎负责"对不对",二者分工**。
> 全链路分四段:**准备(1-3) → 启动(4-6) → 执行循环(7-13) → 收尾产出(14-16)**。

---

## 整体架构:五层 + 一界面

> 平台按职责切成五层,数据**自上而下**流动:用例进 → 规格化 → 智能辅助 → AI 执行 → 出结果代码;
> 五层之上再罩一层**工程化界面**给人用。每层只干一件事、可独立替换。

| 层 | 这一层干什么(大白话) | 输入 → 输出 | 关键技术 / 机制 |
|---|---|---|---|
| ① 输入层 | 把各种来源的用例统一收进来 | Excel/API → TestCase | openpyxl 解析 + 预置条件 LLM 三分类 |
| ② TestSpec 层 | 把人话用例翻译成机器规格 | TestCase → 执行规格+断言 | LLM 一次性翻译 → 结构化 TestSpec/Assertion |
| ③ Page Intelligence | 看懂页面、把模糊词对上真实元素 | 页面 → 业务词↔元素映射 | 词汇表 + 页面扫描(Scanner)+ 预解析 |
| ④ Agent Harness | AI 真正驱动浏览器把用例跑完 | 规格 → 执行过程+判定 | ReAct + 自愈 + 规则引擎断言(核心层) |
| ⑤ 输出层 | 把执行变成可留存的资产 | 执行 → 代码+记录 | BDD 代码生成 + SQLModel 持久化 |
| ★ 工程化界面 | 给人看、给人控的控制台 | — | React + FastAPI(纯 API)+ SSE 实时推送 |

> **一句话记忆**:①收用例 → ②变规格 → ③懂页面 → ④跑起来 → ⑤留资产,**界面**统一操控。
> 下面 16 步,就是这五层在**一条用例**上具体怎么协同跑起来。

## 准备:把"人话用例"翻译成"机器可执行的规格"

| 步骤 | 动作 | 描述 | 关键技术 / 机制 |
|---|---|---|---|
| 1 | 读取 Excel 用例 | 把测试用例读进系统 | openpyxl 容错解析 → 统一 TestCase |
| 2 | 拆解预置条件 | 区分登录态与操作步骤 | LLM 意图三分类(状态/操作/模糊) |
| 3 | 生成执行规格与断言 | 把用例翻译成机器规格 | LLM 翻译 → 结构化 TestSpec + 断言 |

## 启动:为这条用例搭好独立执行环境

| 步骤 | 动作 | 描述 | 关键技术 / 机制 |
|---|---|---|---|
| 4 | 启动独立执行环境 | 每条用例互不干扰 | Subagent 上下文隔离 |
| 5 | 执行登录等前置 | 自动登录或复用登录态 | Hooks + Session/Cookie 复用 |
| 6 | 组装 AI 提示词 | 告诉 AI 该做什么 | 分层 Prompt + Skill 动态注入 |

## 执行循环:AI 一步步操作浏览器(7-13 反复循环)

| 步骤 | 动作 | 描述 | 关键技术 / 机制 |
|---|---|---|---|
| 7 | AI 推理决策 | AI 想好下一步怎么做 | ReAct + StepPlan 状态机 + A11y 树 |
| 8 | 高危操作审批 | 危险操作先人工确认 | Permission 拦截(高危词/prod 锁) |
| 9 | 操作浏览器 | 点击、填写、跳转页面 | playwright-mcp(stdio,绕代理) |
| 10 | 观察页面结果 | 看操作后的页面变化 | A11y 快照 + 截图 + URL 回灌 |
| 11 | 验证预期结果 | 用规则判断对不对 | 规则引擎确定性断言(非 LLM 眼判) |
| 12 | 压缩上下文防卡死 | 控内存、防死循环 | Context Compact + 循环检测护栏 |
| 13 | 失败自动重定位 | 界面变了自动找元素 | Self-Healing 子 Agent(P1→P5) |

## 收尾产出:出结果、出代码、入库

| 步骤 | 动作 | 描述 | 关键技术 / 机制 |
|---|---|---|---|
| 14 | 执行收尾清理 | 退出登录、保存状态 | after_case Hooks |
| 15 | 生成测试代码 | 产出可维护的测试脚本 | CodeGenerator → pytest-bdd Playwright |
| 16 | 入库与汇总报告 | 存结果、出整体报告 | SQLModel 持久化 + Orchestrator 汇总 |

---

## 方案三个核心亮点

| 亮点 | 一句话 | 背后技术 |
|---|---|---|
| 判断前移 | 对错在翻译时定好,执行只做确定比对 | 预期 → 结构化断言 + 规则引擎 |
| 自动自愈 | 界面变化自动重新定位,少人工维护 | 多策略定位(语义角色→视觉) |
| 执行即沉淀 | 跑一次就产出可长期维护的测试代码 | TestSpec + 录制 → BDD 代码生成 |
