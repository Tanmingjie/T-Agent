# 内网用例对接 — 数据规范（v1）

> 本文档由「Web 自动化测试平台」(以下简称**外部平台**)给出,是双方对接的**数据契约**。
> 本文档**只规定数据结构与格式**,不规定接口形态(URL/方法/分页/鉴权等由内网自行决定)。
> 内网测试用例管理平台(以下简称**内网平台**)按本规范**组织并提供数据**;字段对不齐处,
> 由内网在输出前做适配/转换,使其符合本规范。
>
> 对接目标:外部平台用户在【新建测试任务】时,从内网**浏览、筛选、勾选**用例,勾选后按
> 用例 ID 取**详情**并落入自己的执行库。外部平台**只读**,不回写。

---

## 1. 通用约定

- 编码:UTF-8。
- 文本:JSON。
- 时间:unix 秒(整数)。
- 只读:本对接不涉及任何写数据。

---

## 2. 目标用例结构(外部平台最终需要的字段)

外部平台内部只认下面这一个结构,每个内网用例最终被映射成它。内网提供的数据**必须能无损映射到此**:

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `id` | string | 是 | 用例唯一编号。**平台内永久稳定**,作为外部平台关联/去重的主键。不得用会变的行号。 |
| `name` | string | 是 | 用例标题/名称。 |
| `steps` | **string[]** | 是 | 测试步骤,一步一项。自然语言祈使句,**测试数据内联在句子里**(例:`"在用户名输入框输入 zhangsan"`)。**不含** CSS/xpath/定位器/代码,只写"做什么"。 |
| `preconditions` | **string[]** | 否(可空数组) | 前置条件,纯背景说明(例:`"已开通测试账号"`)。 |
| `expected` | **string[]** | 否(可空数组,强烈建议有) | 预期结果。每项须是**可核验的事实**,详见 §4。 |

> ⚠️ **最关键约束**:`steps` / `preconditions` / `expected` 都必须是**字符串数组**。
> 若内网库里存的是一整段文本,需在输出前**拆成数组**,不要把多步骤塞进一个字符串。
> 拆分规则(按换行 / `1. 2. 3.` 编号 / `①②③` 等)由内网决定,但需在交付的映射说明中写明。

**完整示例(单个用例)**

```json
{
  "id": "TC-10231",
  "name": "正常登录-密码正确",
  "preconditions": ["已有可用测试账号 zhangsan"],
  "steps": [
    "打开登录页",
    "在用户名输入框输入 zhangsan",
    "在密码输入框输入 Test@123",
    "点击登录按钮"
  ],
  "expected": [
    "页面跳转到工作台,URL 包含 /dashboard",
    "页面右上角显示用户名 zhangsan"
  ]
}
```

## 3. 两种用途的字段集(数据分层)

同一用例在两种场景下被使用,内网组织数据时可据此分层(供选择时不必带大文本):

| 用途 | 必含字段 | 可选字段 |
|---|---|---|
| **供勾选**(浏览/筛选列表) | `id`、`name` | `module`(模块/目录,供分组筛选)、`tags`(标签)、`updated_at`(unix 秒,供"有更新"提示) |
| **拉详情**(勾选后取全文) | §2 全部 5 字段 | — |

> "供勾选"场景建议不带 `steps`/`expected` 大文本,以减小数据量;但这是性能建议,非强制。

---

## 4. `expected` 格式约束(自动裁决成败关键)

外部平台用**自动化方式**核验预期是否达成,因此 `expected` 每一项必须落在"机器能从页面读到"的事实上:

- ✅ 推荐:可见文字 / 页面文案 / 标题 / 提示语 / URL 片段
  - `"页面显示 提交成功"`、`"列表出现 张三 这一行"`、`"URL 包含 /detail"`
- ❌ 避免:仅靠图标 / 颜色 / 状态灯 / 图片描述,无任何文字依据
  - `"出现绿色对勾"`、`"购物车图标高亮"`、`"按钮变红"`

> 内网若大量存在 ❌ 形式的预期,请在交付的**风险清单**里举例标出。这类用例不是不能接,
> 但自动裁决会偏向判失败,需后续人工补一条"文字依据"或转写。**对接阶段先如实暴露,不要静默丢弃。**

---

## 5. 字段映射(内网填写并交回)

内网把现有用例数据字段对照本规范目标字段,填写如下映射表交回外部平台:

| 目标字段 | ← 内网字段 | 转换/拆分说明 |
|---|---|---|
| `id` | | |
| `name` | | |
| `steps` | | 整段文本?拆分规则=? |
| `preconditions` | | 整段文本?拆分规则=? |
| `expected` | | 整段文本?拆分规则=?是否含 §4 ❌ 形式 |

---

## 6. 交付物清单

1. **数据样例**:≥ 2 条真实用例的 JSON,符合 §2 结构,证明 `steps`/`preconditions`/`expected`
   确为**字符串数组**且内容符合 §2/§4 约束。
2. **字段映射表**(§5)。
3. **风险清单**:`expected` 含 §4 ❌ 形式的用例举例;无稳定 `id` 等问题(如有)。
