> **衔接协议(每个 agent 必守)**:
> 1. 开工前按顺序读:`CLAUDE.md` → `平台化设计草案.md`(方向与已拍板决策,**不重新讨论已决策项**)→ 本文「当前状态」与任务表。

## M1 多租户核心链路(本地双进程形态)

> 原则(草案 §7):**进程边界 = 未来 Pod 边界**。API 与 worker 本地就是两个进程,
> 走 Postgres 任务表(SKIP LOCKED)+ LISTEN/NOTIFY 通信。不许退回进程内起线程的形态。

- [ ] **T-P01 storage 连接串/方言抽象**
  - `DATABASE_URL` env 驱动(缺省仍 sqlite,向后兼容);WAL pragma 等 SQLite 特化按方言分支;JSON 列类型适配(PG 用 JSONB);`Store` 构造不再写死 sqlite 路径。
  - DoD:全量单测 SQLite 下绿;PG 侧单测写好但加「无 PG 则 skip」标记(`DATABASE_URL_TEST_PG` env 存在才跑)。
- [ ] **T-P02 本地 Postgres 落地**(被 PG 二进制阻塞)
  - 解压 zip → `initdb` → `pg_ctl` 启停脚本(`scripts/pg.ps1`,数据目录 gitignore);`asyncpg` 入 requirements;T-P01 的 PG 标记单测真跑绿。
  - DoD:`DATABASE_URL=postgresql+asyncpg://...` 下全量单测绿;README/本文记两地 PG 启停方式。
- [ ] **T-P03 Alembic 迁移引入**
  - baseline 迁移取代 `create_all`(开发期 create_all 可留作 fallback);后续所有 schema 变更走迁移脚本。
  - DoD:空库 `alembic upgrade head` 建出与现 schema 一致的库;单测验证。
- [ ] **T-P04 多租户数据模型**
  - 新表:`Project` / `Version` / `User` / `ProjectMember`(role: admin|tester)。现有 `Suite`/`Run`/`PageVocabularyRow`/词条 加 `project_id`(Suite/Run 再加 `version_id`、启用 `owner`)。Repository 层全部查询套租户过滤。
  - 版本继承:`POST /versions/{id}/clone-suites`(从指定版本拷贝 Suite,显式动作)。
  - DoD:迁移脚本 + repo 单测(跨项目不可见、版本内可见)。
- [ ] **T-P05 鉴权层 AuthProvider + 三角色 RBAC**
  - `AuthProvider` 接口(M4 换 IDaaS 只换实现);一期本地账号(或 header 透传用户名)。三角色:平台管理员/项目管理员/测试人员;FastAPI 依赖注入做路由级权限。
  - DoD:权限矩阵单测(非成员 403、tester 不能改项目配置等)。
- [ ] **T-P06 项目级 LLM 配置**
  - `ProjectLLMConfig` 表(model/api_base/api_key,**api_key 加密落库**:`EncryptedStr` 类型 + 平台密钥 env);`LLMClient` 改 per-run 构造(执行路径从项目配置取,CLI 路径保留 env 兜底);连通自检端点(复用 `--check-llm` 逻辑)。
  - DoD:单测(加密往返/取配置构造/CLI 兜底不回归);界面配置页(显尾号)。
- [ ] **T-P07 API 路由租户化**
  - 所有路由套 `/projects/{pid}/versions/{vid}/...` 上下文(或 query 参数,实现时定);suites/execution/results/vocabulary 全改;词汇表挂 project。
  - DoD:API 单测改造全绿;前端 client.ts 同步。
- [ ] **T-P08 任务队列 + worker 双进程**
  - `run_queue` 表(SKIP LOCKED 领取,含心跳/超时回收/重试次数);`scripts/worker.py`(独立进程,复用 `execution_worker` 的隔离模式:独立 loop+独立 Store);API 端 `POST run` 只入队。项目级并发配额在领取处生效。
  - DoD:单测(领取互斥/超时回收/配额);**双进程手工联调跑通一个 run**。
- [ ] **T-P09 进度桥接:LISTEN/NOTIFY → SSE**
  - worker `NOTIFY` 进度事件(沿用现 SSE 事件协议 suite_start/case_start/phase/spec_ready/step_change/case_result/suite_done);API `LISTEN` 转发 SSE;权限审批(permission)改审批工单表(worker 轮询,超时默认拒绝)。
  - DoD:单测 + 前端执行控制台在双进程下实时刷新。
- [ ] **T-P10 产物存储路径抽象**
  - `ArtifactStore` 接口(本地目录实现;M3 换对象存储/PVC 只换实现);截图/generated code 路径按 project/run 分桶;results 路由取图走抽象。
- [ ] **T-P11 前端:登录 + 项目/版本导航壳**
