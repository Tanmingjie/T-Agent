# AI Web 自动化测试 · 业界调研

> **调研目的:** 摸清"业界做得最好的能到什么程度、现在处于什么阶段",提供事实依据。
> **本材料只陈述事实、数据与调研结论,不含倾向性建议;是否引入由决策者判断。**
> **数据截至:2026-06**(活榜单 / 模型分数);一手论文锚点:2025-04。文中每个数字都标注证据来源,**名词与口径解释见文末附录**。

---

## 一、摘要(一页看懂)

**行业现状一句话:AI 做 Web 测试,这一年多能力涨得很快,但有一道鸿沟一直没跨过——它能帮人"干活"(生成、操作、自愈),还不能替人"做判断"(裁定通过/失败)。因此业界做得最好的产品,都不是纯 AI,而是"AI + 人把关"。**

1. **能力大涨是真的。** 通用真实 Web 任务上,纯 AI 独立完成任务的成功率,从 2025-04 的均值 **35.8%**(最强 61.3%)`[一手]`,到 2026 上半年头部模型已冲到 **90%+** `[一手]`;桌面级任务(OSWorld)从 ~12% 升到 **66%**,前沿模型逼近人类水平 `[一手]`。

2. **但"高分"要打折看,"多数"仍很低。** 那 90%+ 多来自**调优系统、且常由 AI 给自己打分**;同一榜单上**多数 AI 仍只能完成约 30%** `[一手]`。能现成用到的,不是榜首那个数。

3. **真正落地的极少。** 70%+ 的公司在试,但**只有 11% 真在生产环境用**`[综述]`;开发者对 AI 输出的信任从 69% 跌到 54%,**67% 的测试人员只接受"必须人工复核"的 AI 测试**`[综述]`。卡点不是 AI 不够聪明,是不够可信。

4. **最难、最值钱的一环是"测试判断"。** 行业共识:**"AI 能生成测试代码,但生成不了测试判断"**`[综述]`。AI 做裁决会产生大量**假阳性**(对的报成错),反而让人不敢用——这是所有问题的根。

5. **口碑最高的产品靠的是"人 + AI"。** 满意度最高的 QA Wolf(用户评分 G2 **4.8**)`[三方]` 本质是**人机协同的托管服务**,不是纯 AI。这是行业对"纯 AI 还不够可靠"的诚实承认。

## 二、业界现状:最佳产品与能做到的程度

### 2.1 赛道四层(各取代表)

| 层 | 代表 | 性质 |
|---|---|---|
| 自然语言 / 自愈测试平台 | **testRigor**、mabl、Functionize、Momentic | 商用,面向 QA 团队 |
| 托管式 AI QA 服务(人 + AI) | **QA Wolf**、Octomind | 卖"可信交付" |
| 开源浏览器 Agent 引擎 | **browser-use**、Skyvern、Stagehand | 执行层框架 |
| AI-native 验证 / 通用 Computer-Use | **TestSprite**、OpenAI Operator、Claude Computer Use | 给 AI 写的代码做验证 / 通用代理 |

### 2.2 关于"最佳产品"——先说清一个前提

**商用产品没有统一的准确率考卷。** 第一章那些 30%、90% 的数字,都是针对**开源 Agent / 基座模型**在**通用公开任务**上跑的学术榜单;**商用 QA 产品不进这些榜**。所以商用"最佳"只能用三类口径衡量,**且都不可横向硬比**:第三方满意度评分 `[三方]`、落地/回本周期 `[三方]`、厂商自报指标 `[厂商]`。

#### A. 满意度口碑最高 —— QA Wolf(G2 **4.8** `[三方]`)

测试即服务——人类 QA 工程师 + AI 帮客户建/维护/跑 E2E 测试。可引用的统计(注意口径):

| 指标 | 数值 | 口径 |
|---|---|---|
| 用户评分 | **G2 4.8 / 5** | `[三方]` 较硬 |
| 落地 / 回本 | **2 个月落地 / 8 个月回本** | `[三方]` |
| 自动化覆盖 | **4 个月做到 80% 用户流程** | `[厂商]` |
| flake(不稳定)率 | 承诺 **zero**、实际 **<1%** | `[厂商]` |
| 客户结果 | **92%** 发布更快、**90%** 消除发布后热修 | `[厂商]` |

除 G2 4.8 外多为厂商自报,缺独立核验,但与第三方高分方向一致。它被视为"最佳",是因为用**人在环**补上"AI 不可靠"、卖"绿就是真绿"的可信度。代价:贵、SaaS 云、**数据要出公司**。

#### B. 代表性 AI-native 产品(对比看)—— TestSprite

**TestSprite 不是公认"最佳",口碑分化**;列出是因为它代表"为 AI 写的代码做验证"这一新方向。西雅图初创、670 万美元种子轮 `[综述]`。

- 厂商自报:把 AI 生成代码的通过率 **42%→93%(一次迭代)** `[厂商]`("验证-修复闭环"口径,非测试准确率)。
- 独立实测(2026):对**复杂业务逻辑/多步流程产生大量假阳性**,自愈只在简单情况有效,仅云端、按量计费贵 → **复杂/可靠场景尚未生产可用** `[三方]`。

> **对内网场景的关键观察:** 商用最佳产品**几乎全是 SaaS 云、数据出门**(QA Wolf、TestSprite、mabl…),强监管/内网/政企场景基本被排除在外。

### 2.3 能做到什么程度(配 §3.1 时间线看)

| ✅ 能稳定做到 | ⚠️ 做不稳定 | ❌ 做不到 |
|---|---|---|
| 生成脚手架/样板代码 | 开放任务的端到端(多数 Agent 仍 ~30%) | 纯 AI 无人值守 |
| 测试数据(含边界) | 复杂业务断言 | 可信地替代人做最终 PASS/FAIL 裁决 |
| **自愈 selector**(应对 UI 变动) | 跨系统数据一致性 | |
| 根因分析辅助 | 大改版后的自愈 | |

> 核心矛盾:**"AI 能生成测试代码,但生成不了测试判断。"** 断言需要理解业务逻辑,这是当前 AI 的短板,也是"假阳性"的根。

## 三、代表性数据

### 3.1 时间线:能力进展(体现时效性;名词见附录)

> 同一个考卷,一年多内数字可从 ~30% 变到 90%+——**这就是为什么过时数据无意义、必须看时间线。**

| 时间 | 数据 | 口径(考卷 + 评分方式) | 来源 |
|---|---|---|---|
| 2024 初 | OSWorld(桌面任务)~**12%** | OS 任务 | `[一手]` Stanford HAI 2026 AI Index |
| 2024 | WebVoyager:browser-use **89.1%** / Skyvern **85.85%** | 少站点·**宽松自动评分**(偏高) | `[一手/厂商]` |
| 2025-01 | OpenAI Operator:WebArena **58.1%**、OSWorld **38.1%** | 模拟站点 / OS | `[一手]` |
| **2025-04** | **真实网站人工评分:均值 35.8%、最强 Operator 61.3%**;多数 Agent(browser-use 30.0%、Claude CU 29.0%)≈ 2024 基线 | **真实站点·人工评分** | `[一手]` arXiv 2504.01382(COLM 2025) |
| 2026-03 | **同一榜单新高 90.53%**(opus-4.6);**但"多数 Agent 仍只完成约 30%"** | 真实站点·人工评 | `[一手]` OSU-NLP / abp 榜单 |
| 2026-04 | 活榜单:头部 **93–97%**(其中 97% 是系统 + **AI 自评**口径) | 真实站点·**评分口径各异、不可横比** | `[一手]` Steel.dev(2026-04-16) |
| 2026 | OSWorld 整体 **66%**;前沿模型 **73–75%**,逼近人类基线 **72–84%** | OS 任务 | `[一手]` Stanford HAI 2026 AI Index |

**读这张表的三个要点:**
1. **进步是真的**——头部从 61%(2025-04)到 90%+(2026)。"纯 AI 永远不行"是错的。
2. **实验室高分有水分**——WebVoyager ~90%,换真实站点+人工评分坍缩到 35.8%;2026 榜首 97% 是"调优系统+AI 自评"。
3. **"头部高分"与"多数仍 30%"并存**——能现成用到的,接近"多数"水平,不是榜首。

### 3.2 落地与信任:理想 vs 现实

| 数据 | 含义 | 来源 |
|---|---|---|
| **70%+ 在试,仅 11% 真在生产跑** | 实验多、真上生产极少 | `[综述]` 2026 |
| 信任 **69%(2024)→54%(2025)**;仅 **33%** 真信任、**3%** 高度信任 | 用得越多越不信 | `[综述]` |
| **61%** 认同"AI 常产出看着对、实则不可靠的结果" | 可靠性是核心痛点 | `[综述]` 2026 |
| **67%** 测试人员只接受"带强制人工复核"的 AI 测试 | 人在环是刚需 | `[综述]` |
| 仅 **25%** AI 项目兑现预期 ROI、**16%** 规模化成功 | 收益真实但难规模化 | `[综述]` |
| 自动化测试(宽口径)ROI **300–500%**、成本降 **78–93%** | 选对场景收益大 | `[综述]` |

> 能力曲线在涨(§3.1),但落地曲线几乎没动(生产仅 11%)。瓶颈不是"够不够聪明",是"够不够可信"。

### 3.3 调研结论:行业当前能 / 不能做到什么(事实陈述)

| 用法 | 当前状态 | 依据 |
|---|---|---|
| 纯 AI 无人值守 + 可信回归门禁 | ❌ 行业做不到 | 多数 Agent 真实任务仍 ~30%;生产落地仅 11%;无团队这么用 |
| AI 自动产出"可信 PASS/FAIL 裁决" | ❌ 行业做不到 | "生成不了测试判断";自评注水/假阳性 |
| AI 生成测试 + **人审断言** | ✅ 已可行 | 67% 只接受带复核;QA Wolf 模式即此 |
| AI 驱动执行(自愈、随机应变)+ 规则确定性裁决 | ✅ 已可行 | 把"判断"放在确定性层,绕开假阳性根因 |
| AI 自愈 selector / 测试数据 / 根因辅助 | ✅ 已可行且有实效 | 从业者公认 `[综述]` |
| 选对场景(固定流程+明确预期)+ 人在环 | ✅ 有真实 ROI | 自动化测试 300–500% ROI(选对场景) |

> **事实边界(非建议):** 行业当前做得到的是 **"AI 提效 + 人/规则把关";做不到的是 "AI 替代判断"。** 截至 2026-06,没有任何产品越过这条线——口碑最佳的 QA Wolf 也是靠人越不过去。

## 附录 A:名词与口径解释

**证据分级标签(全文每个数字都带其一):**

| 标签 | 含义 | 可信度 |
|---|---|---|
| `[一手]` | 原始论文 / 可公开复核的评测榜单(经同行评审或可直接验证) | 高 |
| `[三方]` | G2 / Capterra 等独立评价站的真实用户评分 | 中(有幸存者/营销偏差) |
| `[综述]` | 行业调查 / 媒体综述(二手汇总,口径不一) | 中-低(取方向性) |
| `[厂商]` | 厂商自报(各自定义口径,不可横比) | 低(需交叉印证) |

**"成功率"统计什么:** = 在某考卷的全部任务中,AI **完整、正确做成**的任务占比。例:35.8% = 5 个 Agent 在 300 个真实任务上、由**人工逐条判定"是否真做成"**后的**平均**成功率;61.3% 是其中最强单个。它衡量"能不能把任务做成",不是"判断对错"的能力。(易混:论文里另有 ~85%,那是"自动评委与人类的一致率",不是 Agent 成功率。)

**Benchmark(评测基准)** = 固定任务集 + 评分方法,用来横向比较 AI。**是"考卷",不是产品。** 同一 AI 换考卷,分数可差几十个百分点。

- **Online-Mind2Web** —— 本文主要一手来源。学术基准(俄亥俄州立大学 OSU-NLP),300 任务/136 个**真实**网站、**人工评分**;论文发表于 **COLM 2025(同行评审会议)**,有公开活榜单。**权威性高**,是目前最被认可的"戳破实验室虚高分"的真实基准之一。
- **WebVoyager** —— 早期基准,网站少、任务有捷径、自动评分宽松 → 分数偏高(易"幻觉")。
- **WebArena** —— 自建可复现的**模拟**网站(非真实互联网)。
- **OSWorld / -Verified** —— **操作系统级(桌面)**任务,比纯 web 更难;Verified 是清洗校验版。
- **Computer Use(CU)** —— 靠"看截图 + 操作鼠标键盘"操作电脑的模型(Operator、Claude CU 等)。
- **人类基线** —— 人在同一套任务上的表现,作"天花板"参照。
